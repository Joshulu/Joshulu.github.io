Ruby Classes

Today we're going to talk about computers... well sort of. We're really going to talk about classes. Classes are basically everything you work with in Ruby. Strings, Integers, Arrays, etc. are all Classes.

Computer Class

Okay, after that very breif introduction, we should be ready to talk about a made up class. We're going to call this class Computer. Now, a String class is familiar enough to know that "string" is an instance of the String class. But what does our Computer class create? That happens to be up to us!

Computer Methods

When a class is defined it has an initialize method this tells our Class#new method to create a new instance of our class. Not all classes work this way but most user generated classes do (from what I've seen).

Well what else can we do? What else does a computer do? Well they can search the internet, they can search within themselves, they can store files, retrieve files, and display files (I know they do more than this, but this is a good sample).

So let's use some code now: 


            class Computer
              def initialize
              end
              
              def search_internet
              end
              
              def search_self
              end
              
              def save_file
              end
              
              def retrieve_file
              end

              def display
              end
            end
            
            my_pc = Computer.new
            my_pc.search_internet("Cats")
            my_pc.save_file(cat_picture.gif)
            my_pc.search_self(cat_picture.gif)
            my_pc.retrieve_file(cat_picture.gif)

All the stuff from class Computer until the end at the bottom is the class definition for Computer. All the 'def ...' are our methods we can use on an instance of our class (Just like array.each). After our class definition we have some, what we would call, driver code. It runs our class definition. We're creating my_pc which will behave exactly how Computer class says it will. We can create more Computer objects like your_pc or work_pc. Now, just like #each, we're saying my_pc.do-something, but we can only do what we've defined in our class. (we can write anything in our definitions, they're just there as an example).

Computer Varialbes

In Ruby, when we have a variable x = 5 for example, our code only knows that x = 5 where it was defined. If we look at a couple methods of our Computer class below: 


            def save_file
            @saved_file = "filename"
            x = 5
            end
              
            def retrieve_file
            print @saved_file
            x = 6
            end
We have a couple things going on. We have an x = 5 a @saved_file = "filename" and a x = 6. Normally, our x in our save_file method stays there. So our other x in retrieve_file does not overwrite our first x. In no way do they interfere with one another. However, for our @saved_file variable, we ARE accessing it in our #retrieve_file method. That @ symbol out in front of the variable tells Ruby, "This variable is an instance variable, you can access it anywhere in your class."

Summary

So we have a class Computer, we can call methods on it, and we can share(or not) variables within our entire class definition. That is the basics of Ruby classes and how they're structured and used.
