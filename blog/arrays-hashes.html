What is an Array?

An array is a collection of objects(programming objects). They could be strings, integers, or even other arrays(multi-dimensional arrays).

The kind of information an array holds is entirely up to you. When you define an array, you can put the things you want inside of it. You can add on to it, or remove from it, later at any time. It could be all strings, all integers, a mix of any objects you want it to hold.

They don't just sit there though. We can at any time access the elements we placed inside the array. We do this by using the array's index. array[0] refers to the 1st element we put in the array. array[1] is the second. array[-1] is the last item on the array(no matter the size). array[-2] would be the second to last element.

Arrays are set up to be ordered in the way you added them. If you remove an element from anywhere but the end, all the rest after the removed element shift down one index.

This can get confusing and messy if you're not exactly sure which elements you're moving around and where the rest end up afterwards. But in Ruby you can use another information container...a hash!

What is a Hash?

A hash, like an array, is a collection of objects. Also, like an array, they can be any mixture of objects, and the objects can be access at any time. The elements are ordered by the order you place them in the hash.

However, we don't access them via an index starting from 0 and moving on. We can, in fact, give the objects their own identifier. We call them keys, and the objects we call values.

When to use what?

This section will be fairly short.

We can use them for the most part, interchangeably. However, as I said earlier, using arrays can get messy sometimes. Generally, use a hash when you have specific important information. A contact book would be great. You can use their name as a key, and it points to their phone number. An array would probably be a better place for something like grocery list of sorts, where specificity isn't needed, and items can be removed from anywhere and it won't much matter.
