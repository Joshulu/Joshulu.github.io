Ruby's Map

Alright everyone, we're delving into programming cartography...but not really. I do want to talk about maps though, or rather Ruby's #map enumberable.

How To Make A Map

In order to make a map, we need a few things.

(enum).map { |obj| block }

So what this means is that, for each object in enum we're going to do block to it. Once we block it, we return a new array with the results of that block.

  • enum: what this is asking for is any one object. It could be a number, a string, a range, an array, you name it.
  • obj: This one is sort of a place holder for each individual enum object we iterate through.
  • block: this is the fun part. This is where we get to do all the programming magic. We can say anything we want to happen for each enum. We can double it, add, concat, delete, anything really.

Let's try an example:

(1..10).map { |i| i * 5}

This returns an array with the following values:

$ [5, 10, 15, 20, 25, 30, 35, 40, 45, 50]

If our 1..10 was instead an array of strings, we would return those strings 5 times for each element.

$ ["catcatcatcatcat", "dogdogdogodgodg"]
