Object v. Hash

Now that we've gotten weird personal preference metaphors out of the way...

What are JavaScript Objects and Ruby Hashes? Well they're a collection of things. They can be similar in that a Hash has a series of user created keys, that point to user created values.

There are a couple differences here but for the most part they are quite the same. You can even add on to each of them after defining them. They use different means to do so, but they can be dynamically altered.

The Difference

Now that we've seen how they're the same thing, let's discuss why they aren't the same thing.

Well that's really difficult to do. The main difference I've found is how you call to them.

In JavaScript the object is a variable itself. You would define and call it as: 


            //define
            var object_name = {
              property_1: value_1,
              property_2: value_2,
              property_n: value_n
            };
            
            //get value
            return object_name.property_n;
            
            // => value_n

In Ruby however, the hash object is not a variable. You would define and call it as: 


            //define
            hash_name = {
              :key_1 => value_1,
              key_2: value_2,
              "key_n" => value_n
            }
            
            //get value
            hash_name["key_n"]
            
            // => value_n
The differences here are key_1 and key_2 are both symbols, using different syntax. Both are acceptable ways. The "key_n" string is also a key. Keys can be strings or symbols. When calling a symbol however, it must be written as hash_name[:key_name]. The colon must appear before the key name. It just looks nicer in the hash_name definition, so they allowed it to work that way.

Summary

If I was able to teach you anything about JS objects vs. Ruby hashes, it's that objects are: variables, 'keys' are called properties and can also be strings, and those properties have values. And that hashes are: not variables, keys can be symbols or strings, and keys point to values.
